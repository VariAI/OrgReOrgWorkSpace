# 讨论:产品定义(进行中)

2026-07-23 起。目标:把「agent 里的 IM」从一句愿景,细化成可写进 `design/` `requirements/` 的产品定义。随讨论更新。

## 已明确

- **主要用户 = 企业管理者 / 决策者**。普通员工更多是**上下文来源**,管理者是**价值落点**。
- **核心价值 = 管理决策驾驶舱**:让管理者随时知道**今日任务、项目进展、风险**等,为决策提供依据。
- **痛点**:企微上现有工具(「大圆」?待澄清)满足不了;通用工具**不会为每个企业定制**。
- **差异化机制**:**打通企业内部上下文**,由 agent **自动合成**管理者视图——不逼企业上新系统、不做一次性 bespoke。
- **与平台已有能力同源**:`OrgReOrg` 现给 dev 团队生成的研发进度页 / 日报 / 风险 / 决策看板 / orchestration 看板,就是"从内部上下文合成管理者视图";产品 = 把它**泛化成任意企业的管理者驾驶舱**。
- **自建 IM 的理由(架构演化史)**:企微(融合差 + API 受限 + **聊天记录要付费开通**才拿得到)→ Matrix(不符国内用户 + **历史包袱重**,即已审的 `OrgReOrg_matrix`)→ 现选国内开源 IM(TSDD / 悟空IM,**连通信协议一起自有**)。三步的共同诉求:**要一套完全自控的 IM,才能不受限地拿到内部沟通上下文、把 agent 深织进去**——这正是"打通内部上下文 / 自动合成"的燃料,也是 WeCom/Matrix 都被弃的根因。
- **以「项目」为组织单位(结构性)**:产品不是扁平看板——有第一类的**项目**概念。系统把**和某项目相关的上下文收集起来**,一个**项目 agent 在后台(OrgReOrg)跑**、吃该项目上下文,产出该项目的任务/进展/风险;**管理者驾驶舱 = 跨项目汇总**。接平台已有的 `knowledge/projects/` / `registry/project-status.json`。

## 待澄清(open)

- **OQ-P1 基本已定 → 自动合成**:自建 IM 的全部理由就是不受限地拿内部沟通上下文、由 agent 合成管理者视图;对接外部项目/工单系统是可选增强,非主路。
- **OQ-P5 已定 → A(客户改用这套自有 IM)**:上下文进我们的 IM、agent 深织;代价 = 换 IM 门槛高,产品价值须值得企业切换(MVP 阶段先 dogfood 自己团队规避此门槛)。

## 原型校准(`D:\project\variAI\原型设计`,以原型为准)

原型是用户更完整的产品想法,比上面讨论细。以原型为准,校准:

- **定位**:产品名「**企业 AI 工作空间**」(VariAI)。IM 只是其中的「消息」面。
- **★主角校准**:原型 POV 是**普通员工**(陈晓/后端工程师)的「**统一工作台**」——**每个员工都有一个**,统一处理来自群聊/私聊/数字员工/外部系统的「**统一行动**」。**管理者驾驶舱 = 其中一个角色视图/rollup**,不是唯一主角。(✓ 用户 2026-07-23 确认:主角 = **每个员工**)
- **核心论点(比"驾驶舱"更核心)**:**权限隔离的「数字员工」+ human-in-the-loop**。数字员工是一等组织身份(不是后台 bot):按群授权、默认读不到入群前历史、跨群要审批、**只建议不自动执行、人确认才成事实**;答案按查看者真实权限过滤;全程 `im://` 来源可溯 + 审计(「上下文与来源」查看器)。
- **对象模型(原型真实)**:群(项目/研发/上线/**部门**/**故障**协作群)是聊天容器;**项目/工作主题 = 企业 AI 跨群识别的聚合**(不作独立入口 → 印证 OQ-P6:不绑固定群、AI 归类);同级还有**事件(故障 INC)**、**部门复盘**三种 object flavor。每群一个「**工作目标 + 完成标准**」;「**群看板**」(聊天/看板切换)= 目标+标准+待处理+推进中+**风险与阻塞**+最新结果。工作事项统一为 task/review/permission/suggestion/**decision**/risk。「**交付物**」带验收状态。「**决策**」一等、不可变(ADR 式)。
- **数据来源(校准 OQ-P1)**:主源 = 原生 IM 群聊(`im://`);**外加 12 个外部业务系统**(GitLab/Jenkins/TAPD/CRM/金蝶…)作**只读"事实"供给**——外部事实同样**只建议、人确认**。
- **surface(校准 OQ-P2)**:IM 内 = 消息 + 群看板;另有一等 section:**工作台 / 日程 / 通讯录 / 数字员工 / 应用 / 管理后台**。管理者视图靠 rollup。
- **产品词汇(owner 文档沿用)**:企业 AI 工作空间、数字员工/主要数字员工、统一工作台/统一行动、群看板、工作目标+完成标准、交付物/成果、风险与阻塞、决策、上下文与来源、权限边界、交办、智能建议、回写。
