# 产品形态:企业 AI 工作空间(design owner doc)

> 目标产品设计,据原型 `D:\project\variAI\原型设计`(VariAI 企业 AI 工作空间)与 `../discussions/2026-07-23-product-definition.md` 收敛。产品尚在建设,本文记录**要收敛到的**产品形态;第一版边界见 `../requirements/mvp.md`。
> 命名:原型词汇「数字员工」于 2026-07-23 更名为「**智能体 / Agent**」(理由与决策见 `../discussions/2026-07-23-agent-identity.md`),全部 owner 文档已统一。

## 一句话

企业自用 IM + agent 原生层。**每个员工一个「统一工作台」**,由权限隔离的「智能体」从内部沟通里识别工作、经人确认后汇成可执行、可溯源的行动;管理者得到跨项目的任务/进展/风险视图。

## 用户与角色

- **每个员工**(主角):一个「统一工作台」,统一处理群聊/私聊/智能体/外部系统来的「统一行动」。
- **管理者**:员工工作台之上的 **rollup 视图**——跨项目的任务/进展/风险,用于决策。
- **智能体**:一等组织身份(非后台 bot),按群授权、只建议不自动执行。
- **管理员**:管理后台治理组织/群与项目策略/智能体/权限审计/AI 用量。

## 核心机制(the loop)

群聊(事实源)→ 智能体识别出建议(任务/风险/交付物/决策)→ **人确认** → 进「群看板」+「统一工作台」+「日程」→ **回写**群里,全程 `im://` 来源可溯 + 审计。

## 核心论点(差异化)

**权限隔离的智能体 + human-in-the-loop + 全程可溯源。**

- 智能体按群授权、默认读不到入群前历史、跨群读要审批、外部系统只读只建议。
- 一切由真实员工确认才成事实;答案按查看者真实权限过滤。
- 区别于「企微/钉钉上贴个 bot」(受限、通用、不可溯),也是自建 IM 的理由(见 `../architecture/project-vision.md`)。

## 对象模型

- **企业 / 部门 / 员工**(通讯录、HR 字段)。
- **群**:聊天容器,有类型——项目协作群 / 研发协作群 / 上线协作群 / **部门群** / **故障协作群**。
- **工作主题**:企业 AI **跨群识别的聚合**,三种 flavor——**项目**(phases/进度/目标/风险数)、**事件**(故障 INC)、**部门复盘**。不作独立协作入口,只聚合上下文。
- **工作目标 + 完成标准**:每群一个目标 + 若干完成标准(done/doing/waiting)。
- **统一行动**:归一化的工作事项,kind ∈ task / review(AI 验收建议)/ permission / suggestion / **decision** / **risk**(高/中/低)。
- **交付物 / 成果**:带验收状态(accepted / pending / rejected / missing)。
- **决策**:一等、不可变(ADR 式),变更走「发起变更」、原记录不动。
- **智能体**:编号(agent_no)、owner、capabilities、tools、scope、审计日志。

## Surface / 信息架构

- **消息**(IM):会话列表 + 聊天 / **群看板**切换 + 上下文与来源面板。
- **工作台**:统一行动收件箱 + 我的协作群(含风险汇总)+ 今日日程 + 最近协作。
- **日程**:智能体智能建议 + 任务/事件。
- **通讯录 / 智能体 / 应用 / 管理后台**。
- **企业 AI 统一入口**(⌘K):提问 / 搜索 / 交办,带「回答来源」与「AI 推断≠正式结论」。

## 数据来源

- **主源**:原生 IM 群聊(`im://`)。
- **补充**:外部业务系统(GitLab / Jenkins / TAPD / CRM / 金蝶… 约 12 个)作**只读事实**供给——同样只建议、人确认。

## 与后台平台的关系

智能体在后台 `OrgReOrg`(Harness / framework)跑;上下文按项目归类、按权限投影,复用 `framework` 的 evidence→ontology→projection(`permission_view` / ADR-099)。IM(Server/Web/Admin)是 surface,平台是引擎(见 `../architecture/system-baseline.md`)。

## 非目标(产品级)

- 不做「企微贴 bot」式浅集成(= 被否的 B)。
- 智能体不自动执行敏感动作、不跨权限读、不替人拍板。
